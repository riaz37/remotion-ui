RemotionUI is a **registry-first component kit** for Remotion. Production-ready primitives, scenes, and compositions install as **source you own**. The `remotion-ui` npm package is the CLI, not a runtime dependency for your video code.

If you've used [shadcn/ui](https://ui.shadcn.com), the workflow will feel familiar: search the registry, run `add`, and customize files in your project.

## shadcn for Remotion

| | shadcn/ui | RemotionUI |
|---|-----------|------------|
| Install | `npx shadcn add button` | `npx remotion-ui add fade-in` |
| Config | `components.json` | `remotion-ui.json` |
| Output | React components in your repo | Remotion components in `src/remotion/` |
| Runtime dep | None (copy-paste) | None (copy-paste) |
| Motion | CSS / Radix | `useCurrentFrame()`, `interpolate()`, `spring()` |
| Registry | `registry.json` on CDN | `remotionui.com/r/` |

Both distribute **source**, not a black-box npm UI kit. Use `npx remotion-ui add` — not `npx shadcn add` — so dependencies resolve, compositions register in `Root.tsx`, and `remotion-ui.json` paths apply.

## Philosophy

- **Registry-first**: components ship as JSON registry entries with resolved file trees
- **Source you own, not import**: files land in your project; you control every line
- **Motion primitives + scenes**: low-level enters/exits compose into full-frame layouts
- **CLI ergonomics**: `search`, `diff`, `update`, and custom `build` for your own registry
